## 2. 도입 확정 기능 (로컬 단독 동작)

### A. 군복무 단축 자동 계산  ⭐ 우선순위 1 · 난이도 하
**현황 문제:** `DischargeScreen`이 군종별 복무를 18/20/21개월로 **고정**. 실제로는 입대 시기별 단축이 적용되어 전역일이 부정확.
**구현:**
- `src/constants/serviceTerms.js` 신설 — 입대 연·월 구간별 복무개월 정적 테이블 (정부 고시 기반, 갱신 불필요한 과거값 + 현행값).
- `dateUtils.js`에 `resolveServiceMonths(branch, enlistDate)` 추가 → 단축표 우선, 없으면 현행 기본값.
- `DischargeScreen` 저장 시 `months`를 자동 산출하여 표기("단축 적용됨" 라벨).
- **DB 불필요** — 단축표는 정적 상수.

### B. 다음 계급 · 다음 호봉 카운트다운  ⭐ 우선순위 1 · 난이도 하
**구현:**
- `dateUtils.js`에 `nextPromotion(promotions)` → 다음 진급 계급 + D-Day 반환.
- 호봉(봉급 인상 시점) 카운트다운: `SALARY_GUIDE` 구간 경계까지 D-Day 계산.
- `HomeScreen` 요약 카드에 "다음 계급 D-N / 다음 호봉 D-N" 추가.

### C. 멀티 프로필 (여러 명 등록)  ⭐ 우선순위 2 · 난이도 중
**현황:** 모든 storage 키가 단일 객체. 곰신(여자친구)·부모 수요가 큼.
**구현:**
- storage 구조를 `profiles[] + activeProfileId`로 래핑. 각 프로필이 `{ id, name, photo, branch, enlistDate, ... , leaveRecords, todos }` 보유.
- 마이그레이션: 기존 단일 데이터 → `profiles[0]`로 1회 변환.
- 상단에 프로필 스위처(드롭다운/아바타) 추가, 활성 프로필 기준으로 전 화면 렌더.
- **DB 불필요** — AsyncStorage 배열로 다 처리. (군돌이 최대 12명 → 우리는 5~6명 권장)

### D. 프로필 사진  ⭐ 우선순위 2 · 난이도 하
**구현:**
- `expo-image-picker` 추가, 선택 이미지를 `expo-file-system` 앱 도큐먼트에 복사·경로 저장(로컬).
- `HomeScreen` D-Day 카드 배경/아바타에 사진 표시. 멀티 프로필(C)과 연동.

### E. 일수별 맞춤 응원 메시지  ⭐ 우선순위 1 · 난이도 하
**현황:** `getRandomMessage()`가 phase 무관 완전 랜덤.
**구현:**
- 메시지 풀을 phase별(`done/d3/d7/d30/d100/normal`)로 분류 → `getMessageForPhase(phase)`.
- 첫 실행/마일스톤 달성 시 응원 팝업(모달) 노출. HomeScreen에 이미 있는 phase 로직 재사용.

### F. 간부(부사관·장교) 호봉 지원  · 우선순위 3 · 난이도 중
**구현:**
- 신분 구분 추가: `personnelType: 'enlisted' | 'nco' | 'officer'`.
- 간부는 계급 진급 대신 **호봉(연차)** 기반 표시, 복무 단축/병사 봉급표 비적용 → 신분별 분기.
- `calcRank`/`SALARY_GUIDE`를 신분별 테이블로 확장(정적).

### G. 휴가 캘린더 뷰  · 우선순위 3 · 난이도 중
**현황:** `LeaveScreen`은 리스트만.
**구현:**
- `react-native-calendars`(무료 OSS) 도입 또는 경량 자체 월간 그리드.
- 연가·포상휴가 기록을 날짜에 마킹, 탭 시 상세. 잔여일수 요약은 기존 로직 재사용.

### H. 홈 화면 위젯  · 우선순위 4 · 난이도 상
**구현:**
- 안드로이드 App Widget(네이티브) — Expo Config Plugin + `react-native-android-widget`(무료 OSS).
- 위젯에 D-Day·계급·% 표시, 투명도 옵션. 데이터는 AsyncStorage→SharedPreferences 동기화.
- **DB 불필요**하나 네이티브 빌드 작업 필요 → 별도 스프린트로 분리 권장.

## 4. 권장 실행 순서 (스프린트)

- **Sprint 1 (퀵윈, 로컬·저난이도):** A 군복무 단축 · B 계급/호봉 카운트다운 · E 맞춤 응원 — ✅ 완료
- **Sprint 2 (코어 차별화):** C 멀티 프로필 · D 프로필 사진 — ✅ 완료
- **Sprint 3 (보강):** G 휴가 캘린더 · F 간부 지원 — ✅ 완료
- **Sprint 4 (네이티브):** H 홈 위젯 — ✅ 완료 (실기기 표시는 리빌드 필요)

### 구현 메모
- Sprint1: 전역일 off-by-one 교정(`calcDischargeDate` -1일), `nextPromotion`, `getMessageForPhase`(단계별 응원).
- Sprint2: `storage.js`를 `@profiles_v1` 컨테이너로 재작성(공개 API 유지). 레거시→프로필 자동 마이그레이션에 off-by-one 교정 흡수. `ProfileBar`+`expo-image-picker`(권한 거부 시 설정 이동 안내).
- Sprint3: `LeaveCalendar`(무의존성 월간 캘린더, 리스트/캘린더 토글). `personnelType`(병사/부사관/장교) — 간부는 복무개월·급여 직접입력, 병사 진급/봉급표 미적용. 모든 기존 데이터는 `personnelType` 미지정 시 병사로 동작(하위호환).
- ⚠️ `expo-image-picker`는 네이티브 모듈 → 실기기 사진 기능은 재빌드 필요(미재빌드 시 graceful 안내).
- 온보딩/간부: 설치 첫 화면에서 **병사/부사관/장교** 선택(`OnboardingScreen`). 병사는 기존 로직 무변경, 간부는 계급 선택·복무개월 직접입력·**호봉 카운트다운**(`officerUtils`)·**2025 초임 봉급 참고표 + 직접입력 override**(`militaryRanks`) 적용. 신분은 `data.personnelType` 브리지로 보관(입대정보 입력 전 결정), 기존 사용자는 `militaryInfo.personnelType` 폴백으로 온보딩 미노출. 봉급표는 공개 정적 데이터 기반 **참고·추정치**로 명시(호봉/수당 미반영).
- Sprint4: `react-native-android-widget`로 안드로이드 홈 위젯(`Discharge`). 진입점을 `index.js`로 변경(위젯 태스크 핸들러 등록). `src/widget/`(데이터/UI/핸들러/갱신). 입대정보 저장·프로필 전환 시 `updateDischargeWidget()` 호출. iOS 미지원(위젯), 안드로이드 전용. 실기기 표시는 `expo prebuild`+리빌드 필요(미링크 시 graceful 무시).

신규 의존성(모두 무료 OSS): `expo-image-picker`, `expo-file-system`, `react-native-calendars`, `react-native-android-widget`.

## 5. 핵심 차별점 정리
군돌이 대비 우리는 **서버리스·완전 로컬**이 강점(프라이버시·오프라인). 위 A~H만 도입해도 군돌이의 "혼자 쓰는 군생활 관리" 핵심 가치(전역계산·멀티프로필·위젯·맞춤응원)를 서버 비용 0원으로 커버 가능. 커뮤니티/편지/실시간급식은 서버 비용 발생 영역이므로 의도적으로 제외.
